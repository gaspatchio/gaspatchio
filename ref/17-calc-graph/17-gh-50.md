## Overview

Add the ability to capture and display actual computed values alongside expressions in the calculation graph, similar to Excel's formula auditing features. This would allow users to see both the formula (e.g., `=A1+B1`) and the actual values (e.g., `5+3=8`) for any calculation in the model.

## Motivation

Currently, the calculation graph in debug mode captures:
- The column being calculated
- The expression/formula
- Source location

What's missing is the ability to see the **actual values** flowing through the calculations. This feature would provide:

1. **Excel-like transparency**: See both formulas and values side-by-side
2. **Easier debugging**: Understand exactly why a calculation produced a specific result
3. **Model validation**: Compare against Excel models value-by-value
4. **Better error diagnosis**: When something goes wrong, see the exact values that caused it

## Proposed Implementation

### Enhanced TracedOperation Structure

```python
@dataclass
class TracedOperation:
    """Complete operation with metadata and values."""
    alias: str
    expression: Any  # pl.Expr
    metadata: OperationMetadata
    expected_dtype: Any | None = None
    # NEW FIELDS for value tracking:
    values: dict[int, Any] | None = None  # {time_index: value}
    sample_values: list[Any] | None = None  # First few values for display
    value_shape: tuple[int, ...] | None = None  # (num_rows, vector_length)
    dependencies: list[str] | None = None  # Columns this operation depends on
```

### Implementation Strategy

#### Option A: Post-Execution Value Mapping (Recommended)

After model execution, map the results back to the computation graph:

```python
def enrich_computation_graph_with_values(af: ActuarialFrame, result_df: pl.DataFrame):
    """Map actual values back to the computation graph after execution."""
    
    for operation in af._computation_graph:
        column_name = operation.alias
        
        if column_name in result_df.columns:
            values = result_df[column_name]
            
            # For vector columns (e.g., projections over time)
            if isinstance(values[0], list):
                operation.sample_values = {
                    'first_5_periods': values[0][:5],
                    'last_value': values[0][-1],
                    'shape': (len(values), len(values[0]))
                }
            else:
                # For scalar columns
                operation.sample_values = values[:3].to_list()
```

#### Option B: Live Value Tracking

Track values as calculations happen (more memory intensive):

```python
class ValueTrackingActuarialFrame(ActuarialFrame):
    def __setitem__(self, key: str, value: Any):
        super().__setitem__(key, value)
        
        if self._tracing and self._track_values:
            # Sample execution to capture values
            temp_result = self._df.with_columns(
                self._convert_to_expr(value).alias(key)
            ).select(key).limit(100).collect()
            
            # Store in computation graph
            if self._computation_graph:
                self._computation_graph[-1].values = temp_result[key].to_list()
```

### Excel-Style Output Format

Create formatted output showing both formulas and values:

```python
def format_excel_style_trace(operation: TracedOperation, time_index: int = 0) -> str:
    """Format operation like Excel's formula bar with values."""
    
    # Parse expression to identify column references
    expr_str = str(operation.expression)
    columns_in_expr = extract_column_references(expr_str)
    
    # Build value substitution
    value_expr = expr_str
    for col in columns_in_expr:
        if col in operation.dependency_values:
            value = operation.dependency_values[col][time_index]
            value_expr = value_expr.replace(f'col("{col}")', str(value))
    
    result = operation.values[time_index] if operation.values else "?"
    
    return f"""
    Formula: {operation.alias} = {operation.original_syntax or expr_str}
    Values:  {operation.alias} = {value_expr} = {result}
    """
```

## Usage Examples

### Basic Usage

```python
# Enable value tracking during model run
config = ModelRunConfig(
    directory=Path("models/my_model"),
    mode="debug",
    track_values=True  # NEW
)

# Run single policy
result = run_single_policy(config, policy_id="1")

# Access calculation trace
trace = result.calculation_trace
```

### Viewing Calculation with Values

